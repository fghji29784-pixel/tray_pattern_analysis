# 트레이 공간 패턴 분석기 (Tray Pattern Analyzer)

활성화공정 OCV 데이터에서 **트레이(12×12=144셀) 내부의 온도·ΔOCV 공간 패턴**을 찾아
"이런 패턴을 가진 트레이가 몇 개인지", "온도와 ΔOCV 트렌드가 일치하는지"를
일반인도 이해할 수 있게 요약한다.

## 핵심 원칙
- **랏-트레이 복합키** `(Lot ID, TRAY ID)` 로 모든 트레이별 계산 (다른 랏에 같은 TRAY ID 존재 대비)
- **온도 오류 제거**: T1/T2/T3 중 하나라도 **22℃ 미만**인 셀은 센서오류로 제외
- **튀는 셀 제거**: 트레이 내 이상치 셀(MAD 기준)은 구배 피팅·히트맵에서 제외 → 한 셀이 패턴을 가리는 문제 방지
- **라벨(판정등급)은 참고용**일 뿐 판정 기준으로 쓰지 않음 (순환논리 방지)
- **신뢰도 게이트**: 약한 신호를 억지로 라벨링하지 않음. 대표트레이에 R²·세기를 표기
- Python 3.8 **32bit 호환** (scipy/sklearn 미사용, numpy 직접 구현)

## 측정 시점
OCV1 → **(+2일)** → OCV2 → **(+1일)** → OCV3. 세 시점 온도(T1/T2/T3)를 각각 분석하고,
시간이 지나며 온도·패턴이 어떻게 변하는지까지 본다.

## 입력 데이터 (엑셀 한 행 = 셀 1개)

한 트레이 = 144셀(12×12). `CELL NO_Charge #05` 1~144 가 A01~L12에 대응
(열 = (n−1)//12 = A~L, 행 = (n−1)%12 = 1~12).

### 필수 컬럼 (없으면 분석 불가)
| 컬럼명 | 의미 | 단위/형식 |
|---|---|---|
| `Lot ID` | 랏 식별자 (트레이 복합키의 한 축) | 문자 |
| `TRAY ID_Charge #05` | 트레이 식별자 | 문자 |
| `CELL NO_Charge #05` | 트레이 내 셀 번호(1~144) → (row,col) | 정수 |
| `Delta OCV_Delta OCV #07` | ΔOCV (코드가 OCV1−OCV3와 일치하는지 자동 검증) | mV |
| `OCV_PRIVT OCV #01/02/03` | PRIVT OCV 측정기의 OCV1/2/3 | mV, **소수 3자리** |
| `PRIVT OCV #01/02/03 온도` | 각 OCV 측정 시점의 셀 온도 | ℃ |

### 선택 컬럼 (없으면 해당 분석만 생략, 나머지는 정상)
| 컬럼명 | 활성화되는 기능 |
|---|---|
| `Start Time_PRIVT OCV #01/02/03` | 관계표의 경과시간, 휴지시간 진단 |
| `Cell 위치` | CELL NO 위치매핑 교차검증(일치율 출력) |
| `판정등급` | 결과 색칠(A=양품/E=불량). **판정 기준으로는 미사용** |

### 전공정 컬럼 (있으면 Phase 6 열이력/완화 분석 전체 활성)
| 컬럼명 | 의미 | 특징 |
|---|---|---|
| `OCV #07 OCV` | 방전 직후 OCV(=OCV0). **충방전기에서 측정** | mV, **소수 1자리**(분해능 낮음) |
| `OCV #07 시작시간` | = 방전 종료 시각 (휴지시간 t=0 기준점) | 일시 |
| `DisCharge #07 최저/평균/최고 온도` | 방전 구간 셀 온도(min/avg/max) | ℃, 스윙=max−min 파생 |

### 측정 설비 특성 (해석 시 중요)
- **OCV7**: 방전 직후 **충방전기**에서 그대로 측정 → 완화 전 전압 + 분해능 0.1mV
- **PRIVT OCV1/2/3**: 별도 **OCV 측정기 4대** 중 하나에서 측정(트레이마다 다를 수 있음)
  → `OCV1`과 `OCV3`가 다른 설비면 **설비 핀맵 오프셋**이 ΔOCV에 섞일 수 있음
  → **[27] 공유 지문 탐지**로 이 가능성을 1차 판별
- 향후 분석 강화를 위해 **OCV1/2/3 측정 설비(호기) ID 컬럼**이 있으면 핀맵 캘리브레이션 가능

### 공정 순서 (시간 기준)
```
방전7(7단계: SOC100→30, 前3=정용량 / 後4=정전압)
   └─ 직후 ── OCV7 ── +1일 ── OCV1 ── +2일 ── OCV2 ── +1일 ── OCV3
```
ΔOCV(=OCV1−OCV3)는 OCV1~OCV3 구간(총 3일)의 전압강하. OCV7까지 있으면
완화(OCV7→OCV1)와 자가방전을 분리(4점 완화모델 [22])할 수 있다.

## 실행
```bash
pip install -r requirements.txt
python tray_pattern_analyzer.py --input "데이터.xlsx" --outdir "results"
```

## 패턴 정의 (히트맵 표시 기준)
- **선형 기울기 8방향**: 오른쪽 / 왼쪽 / 위쪽 / 아래쪽 + **대각선** 오른쪽위 / 왼쪽위 / 오른쪽아래 / 왼쪽아래
- **방사형**: 가운데(중앙) 높음 / 바깥(외곽) 높음
- **뚜렷한 패턴 없음(균일)**: 구배 설명력(R²) < 0.15

판정 방식: 트렌드면 `값 ≈ a + b·(좌우) + c·(상하) + d·(중심거리²)`을 맞춘 뒤,
곡률(d) 성분이 선형보다 확실히 크면(마진) 방사형, 아니면 기울기 벡터 **각도**로 8방향 분류.
방향은 표시상 **열 증가=오른쪽, 행 증가=아래쪽**. 실제 설비 방향과의 대응은 현장 확인 필요.

## 결과 읽는 법 (핵심)
- **분산게이트(03)**: 빨강(위치효과)이 작으면 → 위치 보정으로 얻을 게 적음
- **강한패턴 비율**(콘솔 출력, R²≥0.30): 낮으면 그 변수는 공간 신호가 약함 = 패턴 개수 과대해석 주의
- **정렬도(14)**: 0 근처면 온도가 ΔOCV 패턴을 설명 못함 → 온도 보정 무용
- **OCV×온도 상관(16)**: `OCVn↔Tn`(같은 시점) 대각선이 크면 OCV 측정이 온도에 민감. ΔOCV 행이 0 근처면 ΔOCV는 온도 무관

## 분석 결과 요약 (진행 중 · 랏 59JFD292A1 기준)
> 실데이터 실행에서 확인된 잠정 결론. 새 랏·물리검증으로 갱신 예정.

1. **OCV 시점 온도 = 선별 인자로 무용** — ΔOCV와 방향 정렬 ~0, 위치효과 10~13%뿐,
   패턴이 시점마다 88.6% 바뀜(냉각 과도현상). 온도는 **불량핀 색출 진단용**으로만.
2. **ΔOCV 트레이 내 구배는 실재하나 작다** — 강한 트레이 구배 세기 0.16~0.31mV
   (현 오프셋 0.8mV의 25~40%). 경계선 셀 과검을 유발할 수 있는 크기.
3. **구배 탄생 = 첫 완화 구간(OCV7→OCV1)** — [19] 완화 정렬 0.32, OCV7 −0.06,
   방전온도 −0.16(약·역방향). 단 이 구간에 **설비 교체(충방전기→OCV기)가 겹쳐** 완화 vs
   설비오프셋 분리가 필요.
4. **강한 공유 지문은 없음(ΔOCV)** — [27] 개별↔공통 상관 중앙 0.44로 넓게 퍼짐(=트레이 고유).
   공통필드의 "중앙4점" 무늬는 ±0.03mV로 미세(색 확대 착시)하고 저표본 밴드 영향. 신뢰도 z·
   반반 재현성으로 진위 판별 중.
5. **처방 방향**: 온도 보정 X → **트레이별 detrend**(구현) + **완화모델 k·V∞ 분리**(구현) +
   **detrend 후 mode+0.8 재판정**(예정). OCV7+방전스윙 조기선별은 가능성 단계.
6. **미해결/필요**: 저표본 밴드 원인(특정 CELL NO 결측), OCV1/2/3 설비 ID(핀맵 캘리브레이션),
   물리검증 라벨(순환 라벨 탈피).
